# Baseline results

Generated by `./env.sh bench` and `./env.sh probe` on an RTX A4000 with the released
`ng.pt` (torch 2.5.1+cu121). Starting numbers for distillation work.

## Teacher benchmark ([bench_teacher.md](bench_teacher.md))

`dev` = mean |Δ| per action dim vs the 16-step reference from the same noise (i.e. how
much quality you lose by using fewer Euler steps, *before* any distillation).

- **~2× is free**: 8 steps deviates only ~0.001–0.002 from 16 steps — no training needed.
- **4 steps** (dev ~0.005–0.007) is the natural step-distillation target; 1–2 steps
  (dev ~0.02–0.03) is where real training is needed to recover quality.
- bf16 is ~1.5–4× faster than fp32 with negligible extra deviation — use it everywhere.
- Any step-distillation method must beat the free 8-step baseline to be worth it.

## Label probe ([label_stats.md](label_stats.md))

400 chunks / 460k frames from SHARD_0000 (56 games).

- **35% idle frames** (no buttons, sticks in deadzone) — the paper's IDLE filtering is
  real and this quantifies how much of the raw data is dead weight for BC.
- Labels are clean: **0 metadata/parquet row mismatches**.
- Left stick active 46%, right stick only 8% — right-stick signal is sparse, which
  matters for the 4 virtual right-stick buttons in the model action space.
- ~40% of chunks ship an `actions_processed.parquet` (filtered/remapped) alongside raw.
